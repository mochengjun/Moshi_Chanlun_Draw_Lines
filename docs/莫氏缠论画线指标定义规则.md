# 莫氏缠论画线指标定义规则

## 1. 概述

莫氏缠论画线指标基于**层级递推**架构，从最细粒度的 sub-x1 级别逐层向上推导出 x1、x2、x4、x8 级别的H/L转折点。每个高级别的H/L点必然是低级别H/L点的子集。

**级别链路：** sub-x1 -> x1 -> x2 -> x4 -> x8

## 2. 级别体系

| 级别 | Multiplier | 基准回调K线根数 | 说明 |
|------|-----------|----------------|------|
| sub-x1 | 0 | 2根K线 | 最细粒度，所有价格反转的基础 |
| x1 | 1 | baseMinRetraceBars * 1 | 基础级别 |
| x2 | 2 | baseMinRetraceBars * 2 | 从x1推导 |
| x4 | 4 | baseMinRetraceBars * 4 | 从x2推导 |
| x8 | 8 | baseMinRetraceBars * 8 | 从x4推导 |

其中 `baseMinRetraceBars` 由K线周期决定，配置如下表：

| K线类型 | 代码 | x1根数 | x2根数 | x4根数 | x8根数 |
|--------|------|--------|--------|--------|--------|
| 1分钟 | 1 | 4 | 8 | 16 | 32 |
| 5分钟 | 2 | 6 | 12 | 24 | 48 |
| 15分钟 | 5 | 16 | 32 | 64 | 128 |
| 30分钟 | 6 | 8 | 16 | 32 | 64 |
| 60分钟 | 3 | 4 | 8 | 16 | 32 |
| 日K | 10 | 5 | 10 | 20 | 40 |
| 周K | 11 | 8 | 16 | 32 | 64 |
| 月K | 20 | 15 | 30 | 60 | 120 |
| 季K | 21 | 5 | 10 | 20 | 40 |
| 年K | 30 | 10 | 20 | 40 | 80 |

## 3. Sub-x1 级别检测

Sub-x1 是整个层级递推的基础，使用与原 calculateMarkPoints 相同的状态机逻辑，但基准固定为 **2根K线**。

**确认条件（双条件，满足其一即可）：**

- **条件1（价格突破）：** 从L点出发，后续K线最低价跌破前一L点价格 -> 确认H点
- **条件2（K线根数阈值）：** 从临时高点开始回调K线根数 >= 2根 -> 确认H点
- **条件3（价格突破）：** 从H点出发，后续K线最高价突破前一H点价格 -> 确认L点
- **条件4（K线根数阈值）：** 从临时低点开始反弹K线根数 >= 2根 -> 确认L点

## 4. 级别递推规则 (deriveNextLevel)

从前一级别的H/L点序列推导出当前级别的H/L点。核心判断依据是**回调/反弹K线根数**与**当前级别基准K线根数**的比较。

### 4.1 规则: 级别确认 (>= 1.0x)

当回调/反弹K线根数 >= `minRetraceBars` 时，判定为**达到当前级别要求**，确认新的转折点。

当回调/反弹K线根数 < `minRetraceBars` 时，判定为**同级别波动**，不确认新的转折点，将当前段合并到前一段。

**原理：** 级别确认仅依据回调/反弹K线根数是否达到当前级别基准阈值。这确保级别判定稳定一致，且在大趋势转换为小趋势时不会遗漏有效的短期回调段。

**示例：** 日K基准=5根。
- 4根K线回调 -> 不确认（< 5根）
- 7根K线回调 -> 确认为 x1 级别（>= 5根）
- 12根K线回调 -> 确认为 x1 级别（>= 5根，x2级别从x1点推导时会再次判定）

### 4.2 规则: 推动段与回调段（适用于所有级别x1-x8）

在级别递推过程中，确认后的转折点将切换趋势方向。后续的段按推动/回调交替进行：

- **推动段（Impulse Segment）**：从已确认转折点出发的第一段，方向与趋势相同
- **回调段（Retracement Segment）**：推动段之后的逆向修正段

**推动段规则：**

推动段可以在barCount不满足阈值的情况下确认，**只要回调/反弹不破已确认转折点的价格**：
- 从确认的L点出发的推动段（上涨段）：只要后续H点的价格 >= 已确认L点价格，即可确认
- 从确认的H点出发的推动段（下跌段）：只要后续L点的价格 <= 已确认H点价格，即可确认

**回调段规则：**

回调段必须满足正常的barCount阈值要求（>= minRetraceBars）。

**规则统一性：**

此推动/回调确认机制适用于所有级别（x1、x2、x4、x8）。无论级别高低，推动段可以短于阈值，回调段必须满足阈值。这确保了：
- 大趋势中的有效推动段不会因为barCount不足而被错误过滤
- 级别推导的一致性和可预测性
- 各级别间规则的统一简洁

## 5. x1同级别走势规则

x1同级别走势是x2级别推导的基础，定义了一组连续的x1级别高低点形成的完整趋势结构。

### 5.1 x1同级别上涨走势

在向上基本走势中，高点上移且低点上移，构成上涨走势。

**基本要求：**
- 至少包含4个转折点（2个高点 + 2个低点）
- 点序列格式：L1 -> H1 -> L2 -> H2 -> ...

**具体规则：**
- 第一个低点L1是上涨的起始点，也是整个同级别上涨走势的最低点
- 第二个低点L2必须高于第一个低点L1（L2 > L1）
- 第一个高点H1必须高于第二个低点L2（H1 > L2）
- 第二个高点H2必须高于第一个高点H1（H2 > H1）
- 后续高低点按此规律依次类推

**走势端点：**
- LowPoint = 第一个L点（走势最低点）
- HighPoint = 最后一个H点（走势最高点）

**区间极值验证规则：**

对于上涨走势中的四点序列 L1 → H1 → L2 → H2，必须满足以下约束条件：

1. **H点区间极值验证**：H1必须是L1和L2之间所有K线中的最高价点
   - 验证区间：从L1的K线索引到L2的K线索引
   - 如果H1不是区间内的最高价，系统自动重新标注为区间内最高价的K线位置

2. **L点区间极值验证**：L2必须是H1和H2之间所有K线中的最低价点
   - 验证区间：从H1的K线索引到H2的K线索引
   - 如果L2不是区间内的最低价，系统自动重新标注为区间内最低价的K线位置

3. **递归验证**：每一个高低交替都遵守这个规则
   - 对于任意连续的四个点 Li → Hi → L(i+1) → H(i+1)
   - Hi必须是Li和L(i+1)之间所有K线的最高点
   - L(i+1)必须是Hi和H(i+1)之间所有K线的最低点

**验证流程：**
```
1. 遍历点序列中的每个H点
2. 找到该H点前后的L点区间
3. 在区间内搜索真正的最高价K线
4. 如果当前H点不是最高，修正到真正的最高点位置

5. 遍历点序列中的每个L点（非起始点）
6. 找到该L点前后的H点区间
7. 在区间内搜索真正的最低价K线
8. 如果当前L点不是最低，修正到真正的最低点位置

9. 迭代验证直到所有点都满足极值条件
```

### 5.2 x1同级别下跌走势

在向下基本走势中，高点下移且低点下移，构成下跌走势。

**基本要求：**
- 至少包含4个转折点（2个高点 + 2个低点）
- 点序列格式：H1 -> L1 -> H2 -> L2 -> ...

**具体规则：**
- 第一个高点H1是下跌的起始点，也是整个同级别下跌走势的最高点
- 第二个高点H2必须低于第一个高点H1（H2 < H1）
- 第一个低点L1必须低于第二个高点H2（L1 < H2）
- 第二个低点L2必须低于第一个低点L1（L2 < L1）
- 后续高低点按此规律依次类推

**走势端点：**
- HighPoint = 第一个H点（走势最高点）
- LowPoint = 最后一个L点（走势最低点）

### 5.3 父子级别关系（x2与x1）

x2是x1的父级别，x1是x2的子级别。**x2级别的高低点统一使用K线根数阈值从x1点集合中推导，与走势识别相互独立。**

**推导方法：统一K线根数阈值推导**

所有父级别（x2/x4/x8）的点均通过 `deriveNextLevel` 函数从子级别点集合中筛选得出，基于以下规则：
- 回调/反弹K线根数 >= minRetraceBars * multiplier
- 灰色地带确认机制（1x ~ 2x阈值范围内）
- K线区间极值验证

**重要说明：**
- 走势识别（同级别走势）仅用于前端可视化和形态分析
- 走势识别结果不影响级别推导逻辑
- 这种设计支持连续中枢组合（如：上涨走势 + 收敛中枢 + 收敛中枢 + 上涨走势）

**历史兼容说明：**
原有的"走势端点推导"方法（deriveHigherLevelFromTrends）已废弃，仅保留用于向后兼容和测试。新的推导流程为：
1. 使用 `deriveNextLevel` 基于K线根数阈值推导父级别点
2. 使用 `identifySameLevelTrendsWithKlines` 识别同级别走势（仅用于可视化）

### 5.4 x1同级别上涨收敛型中枢盘整走势

收敛型中枢盘整走势是指价格波动逐渐收窄，高点先下移再上移，低点持续上移的形态。

**基本要求：**
- 至少包含6个转折点（3个高点 + 3个低点）
- 点序列格式：L1 -> H1 -> L2 -> H2 -> L3 -> H3 -> ...

**具体规则：**
- 低点序列必须持续上移：L1 < L2 < L3（所有低点依次上移）
- 高点序列呈现"下移-上移"形态：
  - H1 > H2（第一个高点高于第二个高点，即下移）
  - H2 < H3（第二个高点低于第三个高点，即上移）
  - H3 > H1（第三个高点必须高于第一个高点，确保整体向上突破）

**走势端点：**
- LowPoint = 第一个L点（走势最低点）
- HighPoint = 最后一个H点（走势最高点）

**识别优先级：** 收敛型中枢盘整优先于普通上涨走势识别。

### 5.5 x1同级别上涨扩张型中枢盘整走势

扩张型中枢盘整走势是指价格波动逐渐扩大，高点持续上移，低点先上移再下移的形态。

**基本要求：**
- 至少包含6个转折点（3个高点 + 3个低点）
- 点序列格式：L1 -> H1 -> L2 -> H2 -> L3 -> H3 -> ...

**具体规则：**
- 高点序列必须持续上移：H1 < H2 < H3（所有高点依次上移）
- 低点序列呈现"上移-下移"形态：
  - L1 < L2（第一个低点低于第二个低点，即上移）
  - L2 > L3（第二个低点高于第三个低点，即下移）
  - L3 > L1（第三个低点必须高于第一个低点，确保整体结构保持向上）

**走势端点：**
- LowPoint = 第一个L点（走势最低点）
- HighPoint = 最后一个H点（走势最高点）

**识别优先级：** 扩张型中枢盘整优先于普通上涨走势识别，但低于收敛型中枢盘整。

### 5.6 中枢盘整走势与级别推导

**重要说明：** 中枢盘整走势的识别仅用于前端可视化，不影响x2级别点的推导。

**走势识别用途：**
- 前端图表中显示不同类型走势的可视化边界
- 形态分析和统计
- API响应中的 `same_level_trends` 字段

**级别推导机制：**
- x2级别点统一通过K线根数阈值从x1点集合中推导
- 即使x1走势被识别为收敛/扩张型中枢，x2推导仍使用统一规则
- 未升级的收敛中枢不会产生额外的父级别点

**级别升级机制（保留）：**
收敛型中枢可以通过满足升级条件获得 `ParentPoints`，但这些点仅用于：
- 标记走势的关键结构点
- 前端可视化展示
- 不直接参与x2级别推导

### 5.7 走势识别优先级

系统按以下优先级顺序识别x1同级别走势：

1. **收敛型中枢盘整** - 最高优先级（需6+点）
2. **扩张型中枢盘整** - 次优先级（需6+点）
3. **普通趋势走势** - 最低优先级（需4+点）

### 5.8 收敛型中枢级别升级规则

在收敛型中枢盘整走势中，当回调K线根数满足特定比例条件时，自动升级为父级别走势，增加父级别画线。

**升级触发条件：**
- 第一次回调K线根数：H1 到 L2 的K线间隔
- 第二次回调K线根数：H2 到 L3 的K线间隔
- 当第二次回调K线根数 > 第一次回调K线根数 × 2 时，触发级别升级

**父级别点映射规则：**
升级后的走势会生成一组父级别点序列，映射关系如下：
- 父级别L1 ← 子级别L1（收敛中枢的第一个低点）
- 父级别H1 ← 子级别H2（收敛中枢的第二个高点）
- 父级别L2 ← 子级别L3（收敛中枢的第三个低点）
- 父级别H2 ← 子级别H3（收敛中枢的第三个高点）

**示例：**
```
子级别点序列: L1(index=0) -> H1(index=5) -> L2(index=8) -> H2(index=11) -> L3(index=20) -> H3(index=25)
第一次回调: H1(5) 到 L2(8) = 3根K线
第二次回调: H2(11) 到 L3(20) = 9根K线

9根 > 3根 × 2 = 6根 → 触发升级

升级后父级别点: L1(index=0) -> H1(index=11) -> L2(index=20) -> H2(index=25)
```

**x2推导规则：**
当收敛型中枢已升级时，x2级别点从升级后的父级别点序列推导，而不是从原始的走势端点推导。这确保父级别走势在图表上有完整的高低点画线。

### 5.9 Fallback机制

当点序列不满足同级别走势条件时（如点数不足或高低点不满足任何走势规则），系统自动回退到K线根数阈值推导规则。

### 5.10 多级别走势识别扩展

x1级别的同级别走势识别规则（包括普通上涨/下跌走势、收敛型中枢、扩张型中枢）同样适用于x2、x4、x8更高级别。

**统一识别逻辑：**
- 各级别使用相同的走势识别算法
- 走势类型优先级在所有级别保持一致：收敛型 > 扩张型 > 趋势型
- 收敛型中枢的级别升级规则在所有级别同样适用

**简化走势支持：**
- 允许只有2个点（1个H + 1个L）的简单走势
- 这意味着x1、x2、x4、x8可以是同一段走势（只有一个高点和一个低点）
- 简单走势同样遵循父子级别约束规则

**级别递推流程：**
```
sub-x1 → x1 → x1走势识别 → x2 → x2走势识别 → x4 → x4走势识别 → x8
```

每个级别的点推导使用K线根数阈值。走势识别仅用于前端可视化。

**父子级别点继承关系：**
- x2级别的所有H/L点必须是x1级别H/L点的子集
- x4级别的所有H/L点必须是x2级别H/L点的子集
- x8级别的所有H/L点必须是x4级别H/L点的子集
- 仅在收敛型中枢升级时允许例外情况（使用ParentPoints映射）

## 8. API响应扩展

指标计算结果的 `extra` 字段包含以下数据：

```json
{
  "mark_points": [...],        // 所有级别的标注点
  "kline_type": 10,            // K线类型
  "levels": [1, 2, 4, 8],      // 活跃的级别
  "same_level_trends": [       // 所有级别的走势数组（包含x1/x2/x4）
    {
      "type": "up",            // 走势方向 ("up" 或 "down")
      "pattern": "convergent", // 走势模式 ("trend", "convergent", "divergent")
      "multiplier": 1,         // 走势所属级别: 1(x1), 2(x2), 4(x4), 8(x8)
      "start_index": 0,        // 起始K线索引
      "end_index": 15,         // 结束K线索引
      "start_timestamp": "...",
      "end_timestamp": "...",
      "high_point": {...},     // 最高点
      "low_point": {...},      // 最低点
      "points": [...],         // 组成走势的所有点
      "upgraded": true,        // 是否已升级为父级别
      "parent_points": [...]   // 升级后的父级别点序列（仅当upgraded=true时有效）
    }
  ]
}
```

**multiplier 字段说明：**
- `1` - x1级别走势
- `2` - x2级别走势
- `4` - x4级别走势
- `8` - x8级别走势

**pattern 字段说明：**
- `"trend"` - 普通趋势走势（高低点同向移动）
- `"convergent"` - 收敛型中枢盘整（波动收窄）
- `"divergent"` - 扩张型中枢盘整（波动扩大）

**upgraded 字段说明：**
- `true` - 走势已满足级别升级条件，parent_points 包含升级后的父级别点
- `false` - 走势未升级，使用原始端点推导更高级别

**parent_points 字段说明：**
当 `upgraded=true` 时，该数组包含4个点：
1. 父级别L1（对应子级别L1）
2. 父级别H1（对应子级别H2）
3. 父级别L2（对应子级别L3）
4. 父级别H2（对应子级别H3）
